# AdminTab Multi-Server Setup

This project runs three Next.js applications:
1. **Main AdminTab** (port 3001) - Main website with admin features
2. **File Manager** (port 3002) - File management system
3. **PDF Editor** (port 3003) - PDF generation tool

## Setup Instructions

### 1. Install Dependencies

```powershell
# Main project
npm install

# File Manager
cd Filemanager
npm install
npx prisma generate
npx prisma db push
cd ..

# PDF Editor
cd english_editor
npm install
cd ..
```

### 2. Configure Environment

File Manager uses `../public/posts` as shared storage (already configured in `.env.local`)

### 3. Start All Servers

You need to run each server in a separate terminal:

**Terminal 1 - Main AdminTab:**
```powershell
npm run dev
```

**Terminal 2 - File Manager:**
```powershell
cd Filemanager
npm run dev
```

**Terminal 3 - PDF Editor:**
```powershell
cd english_editor
npm run dev
```

## Access URLs

- Main Site: http://localhost:3001
- File Manager: http://localhost:3002 (accessible via admin tab)
- PDF Editor: http://localhost:3003 (accessible via admin tab)

## Shared Storage

Files uploaded via File Manager and PDFs generated by Editor are stored in:
```
AdminTab/public/posts/
```

This folder is accessible from all three applications.

## Troubleshooting

### File Manager database errors:
```powershell
cd Filemanager
npx prisma db push
npx prisma generate
```

### Port conflicts:
Check if ports 3001, 3002, 3003 are available. Change ports in package.json if needed.

### Upload path issues:
Verify `Filemanager/.env.local` has:
```
UPLOAD_PATH=../public/posts
```
